| 항목 | 수치 |
| --- | --- |
| 학습 문맥 길이 | 8,000 타임스텝(기존 대비 최대 1000배) |
| 조립 작업 성능 개선 | 단일 스텝 기준선 대비 +87% |
| 문맥 확장 단독 효과 | 1K→8K 타임스텝 전환 시 +62% |
| 완수한 과제 | 5분·10단계 조립(기존 베이스라인은 전부 실패) |

최근 로봇 파운데이션 모델 대부분은 카메라 한 프레임이나 아주 짧은 이력만 보고 다음 동작을 결정한다. 연구진은 이 한계를 정면으로 겨냥해, 시각-운동 문맥을 8000 타임스텝까지 늘린 RoboTTT(Test-Time-Training Robot Policies)를 제시했다. 이는 기존 최고 수준 정책보다 세 자릿수, 즉 최대 1000배 긴 문맥이다.

문맥을 이만큼 늘리면서도 추론 지연 시간은 늘리지 않았다는 점이 이 연구의 실용적 핵심이다. 그 결과 사람의 시연 영상 한 번만 보고 새 동작을 따라 하는 원샷 모방, 실행 중 정책을 즉석에서 개선하는 능력, 외부 교란에 대한 강건성, 여러 단계로 이어지는 긴 호흡의 작업 수행력이 동시에 열렸다.

연구진은 사전학습 문맥 길이를 늘릴수록 실제 폐루프(closed-loop) 로봇 성능이 꾸준히 오른다는 사실을 이번에 처음으로 관찰했다고 밝혔다. 문맥 길이 자체가 로봇 파운데이션 모델의 새로운 스케일링 축이 될 수 있다는 뜻이다.

## 1. 무엇이 문제인가

시각-언어-행동(VLA) 정책을 포함한 기존 로봇 파운데이션 모델은 단일 스텝 혹은 짧은 이력만 참조해 동작을 정한다. 이런 구조에서는 로봇이 방금 전 상황을 기억하지 못하니, 여러 단계로 이어지는 조립 같은 장기 과제나 예상치 못한 방해에 취약할 수밖에 없다. RoboTTT는 이 문맥 부족 문제 자체를 스케일링 대상으로 삼았다.

## 2. 얼마나 벌어지나 (비교)

실제 로봇 조작 과제에서 측정한 수치 차이는 다음과 같다.

| 비교 조건 | 결과 |
| --- | --- |
| 단일 스텝 문맥 기준선 → RoboTTT(8K 타임스텝) | 전체 성능 +87% |
| 1K 타임스텝 사전학습 → 8K 타임스텝 사전학습 | 동일 모델 기준 +62% |
| 5분·10단계 조립 과제 완수 여부 | RoboTTT만 완주, 모든 베이스라인 실패 |

문맥을 8000 타임스텝으로 늘린 RoboTTT는 단일 스텝 기준선 대비 전체 성능이 87% 개선됐고, 같은 모델을 1000 타임스텝으로만 사전학습했을 때와 비교해도 62% 더 나은 결과를 냈다. 다섯 시간이 아니라 다섯 분짜리 10단계 조립 과제를 끝까지 완수한 것도 RoboTTT뿐이었다.

> **요약**: 문맥 길이를 8000 타임스텝까지 늘리자 로봇이 사람의 시연 영상 한 번만 보고 새로운 동작을 따라 하고, 실행 중에도 스스로 정책을 개선하는 능력까지 갖췄다.

## 3. 어떻게 작동하나

RoboTTT의 핵심은 테스트-타임 트레이닝(Test-Time Training)을 VLA 같은 로봇 파운데이션 모델에 통합한 것이다. 이 시퀀스 모델의 순환 상태는 '패스트 웨이트(fast weights)'로 구성되는데, 이는 학습 중은 물론 추론 중에도 경사 하강법으로 갱신되는 파라미터다. 과거 이력을 이 가중치 공간에 압축해 저장하고, 필요할 때 다시 꺼내 긴 문맥 조건화에 활용하는 방식이다.

학습 문맥 길이를 늘리기 위해 연구진은 시퀀스 행동 강제(sequence action forcing)와 절단된 시간축 역전파(truncated backpropagation through time)를 결합한 학습 레시피를 사용했다. 이 조합 덕분에 문맥을 세 자릿수 배 늘리면서도 추론 지연을 늘리지 않을 수 있었다.

## 4. 실무 의미

> **주의**: 이 수치들은 특정 실제 로봇 조작 과제와 5분·10단계 조립 시나리오에서 측정된 결과다. 모든 로봇, 모든 작업에 동일한 개선 폭이 재현된다고 논문이 보장하지는 않는다.

로봇 파운데이션 모델을 다루는 팀이라면, 모델 크기나 데이터양뿐 아니라 '문맥 길이'도 별도의 스케일링 축으로 고려할 만하다는 게 이 연구의 실무적 시사점이다. 특히 여러 단계로 이어지는 장기 과제, 시연 기반 원샷 학습, 실행 중 자기 개선이 필요한 응용이라면 문맥 확장이 직접적인 성능 레버가 될 수 있다.

## 마무리 — agenwiki의 한줄평

문맥 길이를 늘려도 지연 시간이 그대로라는 조건을 지키면서 87%·62%라는 구체적 개선 폭을 실제 로봇으로 입증했다는 점에서, RoboTTT는 "더 크게"가 아니라 "더 오래 기억하게"가 로봇 정책의 다음 스케일링 축이 될 수 있음을 보여주는 근거 있는 사례다.
