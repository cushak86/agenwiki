OpenEvolve, TTT-Discover 같은 자동 발견(automated discovery) 하네스는 흔히 범용 도구처럼 쓰이지만, 실제로는 아카이브·부모 선택·탐색·예산 배분에 대한 여러 설계 선택이 하나의 레시피로 뭉쳐진 복합 시스템이다. 30개의 예산 매칭 하네스를 12개 모델-문제 쌍, 310만 회 이상의 LLM 롤아웃으로 비교한 결과, 모든 상황에서 보편적으로 우월한 고정 하네스는 없었다. 대신 초기 발견 진행도가 최종 성능을 예측한다는 점을 활용해, 여러 하네스를 동시에 시작하고 약한 시도를 가지치는 적응형 예산 재배분이 고정 하네스보다 나은 결과를 냈다.

| 항목 | 수치 |
| --- | --- |
| 비교한 하네스 | 30개(예산 매칭) |
| 모델-문제 쌍 | 12개 |
| 총 LLM 롤아웃 | 310만 회 이상 |
| 핵심 결론 | 보편적 승자 하네스 없음 |

자동 발견 시스템은 진화적 탐색 같은 방식으로 코드나 알고리즘을 자동으로 개선해나가는 데 쓰인다. 그런데 발견 실행은 비용이 크고 본질적으로 확률적이라, 기존 연구들은 흔히 너무 적은 독립 시행만으로 하네스를 비교해왔다. 이렇게 되면 방법론적 개선과 실행 간 우연한 편차를 구분하기 어렵다는 것이 이 논문의 출발점이다.

저자들은 OpenEvolve 계열의 진화적 탐색과 TTT-Discover 탐색 하네스를 아카이브 운영 방식, 부모 선택 규칙, 탐색 전략, 예산 배분 방식이라는 구성 요소 단위로 체계적으로 분해하고, 반복 시행 통계 분석을 통해 이를 대규모로 재조합·평가했다.

## 1. 무엇을 문제 삼았나? — 너무 적은 시행으로 비교된 하네스

기존 하네스 비교 연구의 약점은 시행 횟수 자체에 있었다. 발견 실행이 비싸고 확률적이다 보니, 하네스 간 우열을 가릴 만큼 충분한 독립 시행을 확보하지 못한 채 결론을 내리는 경우가 많았다. 이 논문은 이 지점을 겨냥해, 하네스를 구성하는 아카이브·부모 선택·탐색·예산 배분 요소를 낱낱이 분해한 뒤 30개의 예산 매칭 조합으로 재구성했다.

## 2. 결과는 무엇을 보여줬나? — 보편적 승자는 없다

12개 모델-문제 쌍에 걸쳐 310만 회 이상의 LLM 롤아웃과 반복 시행 통계 분석을 적용한 결과, 모든 상황에서 일관되게 우월한 고정 하네스는 없었다. 오히려 OpenEvolve 계열의 변형들은 대체로 더 단순한 대안보다 낮은 성능을 보였다.

> **요약**: 하네스 선택은 한 번 정해두는 범용 레시피가 아니라, 문제와 모델에 맞춰 조정해야 하는 하이퍼파라미터로 봐야 한다는 것이 이 연구의 핵심 결론이다.

## 3. 그렇다면 어떻게 골라야 하나? — 적응형 예산 재배분

연구진은 초기 발견 진행도가 최종 성능을 예측한다는 사실을 확인하고, 이를 실제 선택 전략으로 확장했다. 여러 하네스를 예산 매칭 조건에서 동시에 시작한 뒤, 진행이 약한 시도는 가지치고 남은 자원을 더 강한 생존 하네스에 재배분하는 방식이다.

| 접근 | 결과 |
| --- | --- |
| 무작위로 고정한 단일 하네스 | 상대적으로 낮은 성능 |
| 비적응형 하네스 앙상블 | 적응형보다 낮은 성능 |
| 적응형 예산 재배분 | 위 두 기준선을 모두 앞섬 |

> **주의**: "OpenEvolve 계열이 대체로 부진했다"는 결과는 이 연구가 실험한 12개 모델-문제 쌍 범위 안에서의 관찰이며, 모든 상황에 그대로 일반화된다고 단정할 근거는 아니다.

## 4. 실무에서 무엇을 참고해야 하나?

- **Q. 자동 발견 하네스를 도입할 때 가장 먼저 고칠 습관은?**
  - **A.** "이 하네스가 표준이니 그대로 쓴다"는 접근을 버리고, 하네스 선택 자체를 문제·모델에 맞춰 튜닝해야 할 하이퍼파라미터로 취급하는 것이다.
- **Q. 고정 하네스 하나에 예산을 몰아주는 것과 적응형 재배분 중 무엇이 나은가?**
  - **A.** 이 연구에서는 여러 하네스를 동시에 시작해 약한 시도를 가지치는 적응형 재배분이, 무작위로 고정한 단일 하네스와 비적응형 앙상블 모두를 앞섰다.
- **Q. 이 결론을 검증할 자료는 공개돼 있나?**
  - **A.** 저자들은 모델-문제 쌍별 기준 null 분포를 포함한 전체 실행 풀을 재사용 가능한 통계 인프라로 공개했다.

> **팁**: 자체적으로 발견·탐색 파이프라인을 운영한다면, 초반 몇 스텝의 진행도만으로 약한 시도를 조기에 걸러내는 장치를 먼저 넣어보는 것이 적응형 재배분의 가장 간단한 출발점이다.
