| 전체 공격 실행 횟수 | 260회 |
| 전체 성공률 | 95.0% |
| 테스트 환경별 성공률 | IoTGoat 94.5% / Metasploitable2 96.7% |
| 평균 실행 시간 | 대부분 2분 미만 |

IoT(사물인터넷) 기기는 하드웨어 제약, 오래된 펌웨어, 취약한 기본 설정 때문에 구조적으로 보안에 취약한 경우가 많다. 이 논문은 LLM 기반 에이전트가 침투 테스트나 CTF(사이버 보안 대회) 환경에서 성과를 보여온 흐름을, IoT라는 특수한 영역에 적용한 첫 사례로 VEXAIoT(Vulnerability EXploitation using AI Agents)라는 멀티에이전트 프레임워크를 제시한다.

VEXAIoT는 취약점 탐지를 담당하는 에이전트와 공격 실행을 담당하는 에이전트로 구성돼, 정찰부터 공격 순서 계획, 실제 공격 실행까지를 자동으로 수행한다. 연구진은 이를 IoTGoat와 Metasploitable2라는 통제된 테스트 환경에서, OWASP가 정의한 IoT 취약점 항목에 대응하는 10가지 공격 시나리오로 검증했다.

> **주의**: 이 연구는 IoTGoat·Metasploitable2 같은 통제된 실습용 환경에서 진행된 보안 연구이며, 실제 서비스 대상 공격 방법을 안내하는 것이 아니다. 목적은 보안 테스트 자동화의 가능성과 한계를 평가하는 데 있다.

## 1. 무엇을 만들었나

VEXAIoT는 두 종류의 에이전트로 구성된다. 하나는 취약점 탐지 에이전트로 정찰과 취약점 식별을 맡고, 다른 하나는 공격 실행 에이전트로 공격 순서를 계획하고 실제 공격 도구를 실행한다. 두 에이전트는 LLM 기반 추론과 기존 공격 보안 도구를 결합해 동작한다. 평가는 IoTGoat와 Metasploitable2라는, 보안 연구·교육 목적으로 만들어진 통제된 취약 환경에서 이뤄졌으며, 공격 시나리오 10가지는 OWASP IoT 취약점 목록에 대응하도록 설계됐다.

## 2. 얼마나 성공했나 — 환경별 비교

| 테스트 환경 | 성공률 |
| --- | --- |
| IoTGoat | 94.5% |
| Metasploitable2 | 96.7% |
| 전체(260회 실행 평균) | 95.0% |

일부 시나리오에서는 성공률이 최대 100%에 달했다고 보고됐다. 연구진은 토큰 사용량(비용)도 낮은 편이었고, 대부분의 공격에서 평균 실행 시간이 2분 미만이었다고 밝혔다. 즉 성공률뿐 아니라 비용·속도 측면에서도 자동화된 공격 실행이 상당히 효율적이었다는 뜻이다.

## 3. 왜 이런 성공률이 나오나

두 에이전트의 역할 분리가 핵심 구조다. 탐지 에이전트가 정찰을 통해 취약점을 특정하고, 실행 에이전트가 그 결과를 바탕으로 공격 순서를 계획해 실행한다. 이 파이프라인이 OWASP IoT 취약점 목록이라는 이미 잘 알려진 취약점 유형에 맞춰 설계됐다는 점도 성공률에 영향을 준 것으로 보인다. 다만 논문 초록은 왜 특정 시나리오가 더 어려웠는지에 대한 세부 원인 분석까지는 제시하지 않는다.

## 4. 실무 의미 — 방어 관점에서 무엇을 봐야 하나

이 결과가 실무에 주는 메시지는 공격 자동화 그 자체보다, IoT 보안 점검을 자동화할 수 있는 잠재력 쪽에 가깝다. 연구진도 이 프레임워크를 "LLM 기반 에이전트가 IoT 취약점 평가와 공격적 보안 워크플로를 자동화할 수 있음을 보여준다"는 취지로 소개한다. 반대로 보면, 알려진 취약점 유형에 대해 자동화된 공격이 이 정도 속도와 성공률로 이뤄질 수 있다는 사실은 방어 측에도 경고가 된다 — 오래된 펌웨어와 기본 설정을 방치한 IoT 기기는 자동화된 도구에 의해 빠르게 발견되고 뚫릴 수 있는 시대라는 뜻이다.

> **팁**: IoT 기기를 운용한다면 기본 계정·펌웨어 업데이트 같은 기초 보안 위생을 자동화 도구의 공격 대상이 되기 전에 먼저 점검하는 편이 안전하다.

## 마무리 — agenwiki의 한줄평

VEXAIoT는 LLM 에이전트가 침투 테스트를 CTF 밖 실제 IoT 환경으로 확장할 수 있음을 통제된 실험으로 보여준 연구다. 공격 자동화의 성숙도만큼이나, 이제는 방어 측의 보안 점검 자동화도 같은 속도로 따라가야 한다는 신호로 읽는 게 맞다.
