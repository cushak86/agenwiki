| --- | --- |
| 검증 대상 | 개념 기반 설명 AI(XAI) |
| 평가 데이터 | 망막 안저(fundus) 영상 |
| 신뢰도 평가 축 | 5가지 |
| 최고 서로게이트 충실도 | R² = 0.8503 |

이 모델이 왜 이런 판단을 내렸는지를 사람이 이해할 수 있는 개념(concept) 단위로 설명해주는 개념 기반 설명 가능 AI(XAI)가 늘고 있다. 그런데 그 개념 설명 자체가 얼마나 믿을 만한지는 자동으로 검증되지 않는다는 문제가 있었다. 연구진은 이 공백을 메우기 위해 ConceptSMILE이라는 감사(auditing) 프레임워크를 제안했다.

개념 기반 설명은 의료 영상처럼 판단 근거가 실제 의사결정에 영향을 주는 분야에서 특히 널리 쓰인다. 하지만 설명이 그럴듯해 보인다는 것과 실제로 모델의 판단 근거와 일치한다는 것은 다른 문제다. ConceptSMILE은 이 간극을 정량적으로 측정하려는 시도다.

이 글에서는 ConceptSMILE이 개념 설명을 어떻게 검증하는지, 망막 영상 실험에서 서로 다른 개념 추출 방식(MedSAM 기반 vs VLM 기반)의 신뢰도가 어떻게 갈렸는지를 정리한다.

## 1. 무엇이 문제인가 — 개념 설명은 왜 자동 검증이 안 되나

기존의 설명 가능 AI 기법인 SMILE은 픽셀·영역 단위의 어트리뷰션(attribution)을 다뤄왔다. 반면 최근 늘어난 개념 기반 설명은 혈관이 있다, 출혈이 보인다 같은 사람이 이해하기 쉬운 개념 단위로 판단 근거를 제시한다. 문제는 이런 개념 단위 출력이 실제로 모델 내부 판단과 얼마나 일치하는지 확인할 자동화된 수단이 없었다는 점이다. ConceptSMILE은 SMILE을 대체하는 것이 아니라, SMILE의 섭동(perturbation) 기반 논리를 개념 단위 설명까지 확장한 감사 계층이다.

## 2. ConceptSMILE은 어떻게 검증하나

ConceptSMILE의 작동 방식은 다음과 같다.

- 입력 이미지의 특정 영역을 섭동(perturb)시킨다.
- 그에 따른 개념 반응(concept-response)의 변화를 측정한다.
- 위치(locality) 가중치를 적용한다.
- XGBoost 서로게이트 모델을 학습시켜 국소적인 개념 행동을 근사한다.

이렇게 얻은 신뢰도는 어트리뷰션 정확도, 서로게이트 충실도, 충실성(faithfulness), 안정성(stability), 일관성(consistency) 다섯 축으로 평가한다. 모델 자체가 아니라 설명이 믿을 만한가를 독립적으로 감사하는 층을 추가하는 셈이다.

> **요약**: ConceptSMILE은 개념 설명을 다섯 가지 신뢰도 축(정확도·충실도·충실성·안정성·일관성)으로 쪼개서 검증하는 감사 프레임워크다.

## 3. 얼마나 다른가 — MedSAM 개념 vs VLM 개념

연구진은 망막 안저 영상에서 두 가지 방식으로 뽑은 개념을 비교했다. 하나는 의료 영상 분할 모델 MedSAM에서 뽑은 시각적 개념이고, 다른 하나는 비전-언어 모델(VLM) 기반의 의미적 개념이다.

| 구분 | MedSAM 기반 개념 | VLM 기반 개념 |
| --- | --- | --- |
| 공간 어트리뷰션 | 더 강함 | 상대적으로 약함 |
| 서로게이트 충실도(R²) | 0.8503 (최고) | 상대적으로 낮음 |
| 혈관(vessel) 충실성 | 상대적으로 약함 | 더 강함 |
| 특정 아티팩트 조건 안정성 | 상대적으로 약함 | 더 강함 |

즉 한쪽이 모든 면에서 우월한 것이 아니라, 신뢰도 축마다 우열이 갈렸다. MedSAM 기반 개념은 공간적으로 어디를 보는지 더 정확했고 서로게이트로 근사했을 때 가장 충실했지만(R²=0.8503, 가중 R²=0.8465), VLM 기반 개념은 혈관에 대한 충실성과 특정 아티팩트 조건에서의 안정성이 더 높았다.

> **주의**: 개념 기반 설명이니까 믿을 만하다는 가정은 위험하다. 같은 과제에서도 개념을 뽑는 방식(MedSAM vs VLM)에 따라 신뢰도의 성격 자체가 달라졌다.

## 4. 실무 의미 — XAI 신뢰도를 어떻게 확인해야 하나

의료 영상처럼 고위험 분야에 개념 기반 XAI를 도입한다면, 개념 추출 방식 하나를 정하고 끝내기보다 ConceptSMILE류의 감사 층을 별도로 두고 다섯 가지 신뢰도 축을 나눠 점검하는 편이 안전하다. 특히 어떤 개념 추출 방식이 자신의 데이터·과제에서 어떤 축(공간 정확도, 충실도, 안정성 등)에 강한지는 실험해보지 않으면 알기 어렵다는 것이 이번 결과가 주는 실무적 시사점이다.

> **팁**: 개념 기반 설명을 도입할 때는 정확도만 보지 말고, 충실도·안정성처럼 서로 다른 신뢰도 축에서 어떤 개념 추출 방식이 강점을 갖는지 별도로 감사해보는 것이 좋다.

## 마무리 — agenwiki의 한줄평

개념 기반 설명 AI는 사람이 이해하기 쉽다는 장점이 있지만, 이해하기 쉬운 것과 믿을 수 있는 것은 다른 이야기다. ConceptSMILE처럼 설명 자체를 감사하는 계층을 두는 접근은, 특히 의료처럼 판단 근거가 중요한 분야에서 앞으로 표준적인 점검 절차가 될 가능성이 크다.
