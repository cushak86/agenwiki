# agenwiki 커스텀 도메인 도입 검토 브리프

**작성:** SEO팀 | **작성일:** 2026-07-18
**대상:** https://agenwiki.vercel.app/ (Next.js 14 / Vercel / 한국어 AI 지식백과, 운영 8일차)
**목적:** 사장 도메인 구매 의사결정용. 애드센스 하드블로커 "소유 최상위 도메인 부재" 해소가 핵심 동기.

## 요약
- 커스텀 도메인은 애드센스 하드블로커 1건을 직접 해소. **사이트가 어려(8일) 색인이 거의 없는 지금이 이전 최적기.** 미룰수록 손해.
- Vercel 연결·SSL 무료. 지출은 도메인 연간 등록비뿐.
- TLD 추천: **`.com`** — 연 약 $10~15(≈1.4~2만원). 최저·안정 갱신비, 등록 제한 없음, 최고 신뢰도.
- 결정 사항은 "어떤 도메인명을 살지" 하나.

## 1. 후보 (실제 선점 여부는 구매 시 레지스트라 실시간 조회 필요)
| 후보 | 성격 | 등록비/년 | 비고 |
|---|---|---|---|
| **agenwiki.com** | 범용 글로벌 표준 | ~$10~15 | 제한 없음, 갱신비 안정 — 추천 |
| agenwiki.dev | 개발자·Google gTLD | ~$12~15 | HTTPS 강제(되돌리기 불가), 국내 인지도 낮음 |
| agenwiki.ai | AI 프리미엄 | ~$65~73 (갱신 $72~85) | 브랜드 최고, 매년 고비용 |
| agenwiki.co.kr | 국내 기업용 | ~$22~30 | 등록 자격 요건 가능, 글로벌 확장 불리 |
| agenwiki.net | .com 대체 | ~$10~15 | .com 선점 시 차선 |

애드센스 관점: 등록형 TLD면 어느 것이든 블로커 해소. 확장자 선택은 브랜드·비용·확장성 문제.

## 2. Vercel 연결 (공식 문서 기준)
1. Vercel → 프로젝트 → Settings → Domains → Add
2. DNS: apex는 A `76.76.21.21`, www는 CNAME `cname.vercel-dns.com`, 또는 네임서버 교체
3. SSL 자동 발급(수 분), 무료
4. 전파: 일반 레코드 빠름, 네임서버 교체는 24~48h
- 권장: **외부 저가 레지스트라(Cloudflare 원가·마크업 0) 구매 → Vercel 연결.** 저렴 + 호스팅 락인 회피. (.ai는 Cloudflare 미취급 → Porkbun/Namecheap)

## 3. 서브도메인→커스텀 이전 SEO 영향 (가장 중요)
- 표준 절차: 301 영구 리다이렉트(180일+ 유지), GSC "주소 변경 도구"(구·신 속성 모두 인증 필요), 새 속성 등록 + sitemap 재제출. Google 재색인 최대 180일.
- **왜 지금이 유리한가**: 이전 비용은 옮길 색인 자산 크기에 비례하는데 지금은 거의 0.
  1. 잃을 색인 자산 없음(8일차) 2. 301 매핑 부담 최소 3. 180일 재색인 리스크를 무트래픽 시점에 소진 4. 처음부터 최종 도메인으로 색인 5. 애드센스 타임라인과 맞물림
- 유의: vercel.app 공유 도메인의 전역 301 완전 제어 가능성은 Vercel 설정 확인 필요(단 현 색인 미미로 영향 경미).

## 4. 권고

## 승인 후 개발팀 전달
1. Vercel 신 도메인 추가 → primary, vercel.app은 301
2. next.config.mjs·canonical·sitemap.xml·robots.txt 신 도메인 일괄 교체
3. GSC 신 속성 등록·인증 → 주소 변경 도구 → sitemap 재제출
4. OG/메타·구조화 데이터 URL 하드코딩 점검
