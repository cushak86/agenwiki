# agenwiki 콘텐츠 스타일 가이드 (정본)

> 이 문서는 agenwiki의 모든 장문 콘텐츠(guides·news)가 따르는 작법 규칙이다.
> 프롬프트(a_model_*.md)와 사람 편집자 모두 이 규칙을 기준으로 삼는다.
> 목표: "훑어도 핵심이 전달되고, 읽으면 관점이 남는" 글.

## 0. 한 줄 원칙

숫자를 앞세우고, 독자의 질문 순서대로 답하고, 마지막에 편집자의 관점을 남긴다.
과장하지 않는다. 근거(초록·소스) 밖의 사실을 만들지 않는다.

## 1. 제목 공식 — 뉴스형 선언 + 대시 부제

형식: `<사건/발견을 선언하는 문장> — <핵심 수치 또는 의미를 담은 부제>`

- 앞부분은 "무슨 일이 있었는지"를 명사·서술형으로 단언한다.
- 대시(—) 뒤에는 독자가 얻는 핵심 수치나 의미를 압축한다.
- 막연한 수식어("놀라운", "혁신적") 금지. 구체 수치·개념으로 대체.

예시:
- `양자화된 LLM은 겉보기엔 같다 — 정확도가 놓치는 판단의 어긋남`
- `2비트까지 압축해도 점수는 그대로 — 무너지는 건 어텐션의 쿼리·키다`
- `긴 컨텍스트 RAG의 함정 — 문서 100개를 넣자 정답률이 오히려 떨어졌다`

## 2. 오프닝 — 스탯 라인 + 리드

### 2-1. 스탯 라인 (핵심 시각 앵커)

리드 문단 바로 위 또는 아래에, 핵심 수치 3~4개를 표로 배치한다.
MDX 안전을 위해 표준 마크다운 표 문법만 사용한다(중괄호·JSX 금지).

```
| 항목 | 수치 |
| --- | --- |
| 대상 모델 | 5종 |
| 양자화 범위 | 8→2비트 |
| 핵심 지표 | 정답 일치도 |
| 급변 구간 | 저비트폭 |
```

- 각 셀은 "큰 숫자 + 짧은 라벨" 감각으로. 문장 금지.
- 수치가 3개 미만이면 표를 생략하고 리드로만 연다.

### 2-2. 리드 (2~3문단)

- 1문단: 무슨 일/무슨 발견인가 (사실).
- 2문단: 왜 중요한가 — 독자·업계 맥락.
- 선택 3문단: 이 글에서 무엇을 알게 되는가.
- 문단은 짧게(3~5문장). 첫 문장에 핵심을 둔다.

## 3. 본문 — 번호 H2 구성 공식

본문은 번호를 붙인 H2 4~5개로 구성한다. 순서는 독자의 질문 순서를 따른다:

1. **무엇인가** — 개요·정의
2. **얼마나인가 (비교)** — 벤치마크·전후 비교. 비교 데이터가 있으면 반드시 표로.
3. **어떻게 작동하나** — 메커니즘·원리 해설
4. **실무 의미** — "이게 당신에게 의미하는 것", 주의점·적용법

예: `## 1. 무엇이 문제인가`, `## 2. 얼마나 벌어지나 (비교)`, `## 3. 어텐션은 어디서 무너지나`, `## 4. 배포 전에 무엇을 봐야 하나`

### 본문 장치

- **비교 테이블**: A vs B를 2열로. 근거 데이터가 있을 때만(없는 수치를 만들지 않는다).
- **콜아웃(blockquote)**: 핵심을 blockquote로 강조한다.
  - `> **요약**: 한 문장 핵심.`
  - `> **주의**: 오해·리스크 경고.`
  - `> **팁**: 실무 적용 한 줄.`
- **짧은 문단·불릿**: 스캔 가능성 우선. 한 문단 한 생각.
- **숫자 선행**: 본문에서도 막연한 서술보다 구체 수치를 앞에.

## 4. 마무리 — 고정 섹션

본문 마지막 H2는 항상 아래로 고정한다:

```
## 마무리 — agenwiki의 한줄평
```

- 단순 요약이 아니라 편집자 관점의 총평 1~3문장.
- "그래서 이 소식/연구를 어떻게 받아들여야 하는가"를 단언한다.
- 근거 밖 예측은 삼가되, 관점은 분명히.

## 5. 길이

- 목표 1800~3000자(한글, 공백 포함). 현행 ~1000자의 2~3배.
- 근거(초록·소스)가 얇아 늘리면 사실을 지어내야 하는 경우엔 늘리지 말고,
  그 사실을 Caveat로 남긴다(가짜 분량 금지).

## 6. 매력 장치 (권장)

- 숫자 선행 / 구체 예시 / 독자 이익 관점("이게 당신에게 의미하는 것").
- 첫 문장 후킹: 첫 문단 첫 줄에서 핵심을 던진다.
- 관점 있는 마무리로 재방문 이유를 남긴다.

## 7. 금지 사항

- 본문 중괄호 `{ }` 사용 금지 (MDX가 JSX 표현식으로 해석 → 빌드 오류).
  변수 예시는 대괄호 `[변수]`로 표기.
- 원문(초록·기사) 문장 verbatim 복제 금지 — 한국어로 재표현.
- 초록·소스 밖의 사실·수치·인용 생성 금지.
- 마케팅·과장 톤 금지("업계를 뒤흔든" 류). 담백한 ~다/~습니다체.

## 8. 출처 블록 (고정)

본문 맨 끝에 출처 식별 블록을 둔다. 본문 표현으로 재사용하지 않는다.

- arXiv: `> 출처: arXiv:<id> <url>`
- 블로그·기사: `> 출처: <매체명> — <url>`

## 9. 커버 이미지 규칙

- frontmatter에 `cover: "/images/covers/<slug>.svg"`를 지정한다(값은 slug와 일치).
- SVG 커버 규격:
  - 캔버스 1200×630.
  - 다크 그라데이션 배경 + 포인트 컬러 악센트 1색.
  - 제목 키워드 2~4단어를 큰 타이포로(전체 제목 아님, 핵심어만).
  - 카테고리 라벨(상단 소형).
  - 하단에 `agenwiki` 워드마크.
  - 한글은 시스템 폰트 스택(`system-ui, 'Malgun Gothic', sans-serif`).
  - 외부 리소스(폰트 CDN·이미지) 참조 없음 — self-contained.
- 템플릿: `scripts/pipeline/cover_template.svg` ({TITLE_LINE1}/{TITLE_LINE2}/{CATEGORY} 치환).
- 개념 설명에 도움이 되면 본문 중간에 인라인 SVG 다이어그램 1개를 권장(선택).
  다이어그램도 self-contained·중괄호 금지.

## 10. frontmatter (스키마 준수 — 필드 추가·변경 금지)

`title`, `slug`, `description`, `category`, `tags`, `publishedAt`, `updatedAt`,
`author`, `cover`(선택), `draft`. 스키마(lib/content.ts) 정의를 따른다.
