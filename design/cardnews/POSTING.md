**캡션:**
```
모델이 "기억"만으로 답하다 헛소리를 한다면? 🧐

RAG(검색 증강 생성)는 답하기 전에 먼저 "찾아보게" 만드는 구조입니다.
그래서 최신 정보나 우리 회사 문서 기반으로 근거 있는 답을 줄 수 있죠.

이 글에서 정리했어요 👇
· RAG 3단계: 문서 준비 → 검색 → 생성
· 언제 RAG를 쓰면 좋은가 (최신성·사내문서·근거 필요할 때)
· RAG의 한계 (검색 품질 = 답변 상한선)
· RAG vs 파인튜닝, 뭘 선택할까

전체 글은 프로필 링크에서 👉 "RAG란 무엇인가"
🔗 agenwiki.vercel.app/guides/what-is-rag
```

**이미지 순서:** `set6/01-cover` → `02-body-five-axes` → `03-body-open-source-wins` → `04-body-api-wins` → `05-body-three-questions` → `06-cta`

**캡션:**
```
직접 모델을 돌릴까, API로 부를까? 🖥️

· 오픈소스가 유리할 때 (데이터 외부 반출 불가·깊은 커스터마이징·대량 사용)

구조적인 트레이드오프만 정리했습니다.
```
