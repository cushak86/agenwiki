```
챗봇한테 물어만 봤지, "일"을 시켜본 적 있나요? 🤖

AI 에이전트는 답변 하나로 끝나지 않습니다.
목표를 주면 스스로 계획하고, 도구를 쓰고, 결과를 검증하며 일을 끝까지 밀어붙이죠.

## 세트 2 — "RAG, 이름은 들어봤는데 정확히 뭔가요"

**이미지 순서:** `set2/01-cover` → `02-body-3-steps` → `03-body-when-to-use` → `04-body-limitations` → `05-body-vs-fine-tuning` → `06-cta`

**캡션:**
```
모델이 "기억"만으로 답하다 헛소리를 한다면? 🧐

RAG(검색 증강 생성)는 답하기 전에 먼저 "찾아보게" 만드는 구조입니다.
그래서 최신 정보나 우리 회사 문서 기반으로 근거 있는 답을 줄 수 있죠.

이 글에서 정리했어요 👇
· RAG 3단계: 문서 준비 → 검색 → 생성
· 언제 RAG를 쓰면 좋은가 (최신성·사내문서·근거 필요할 때)
· RAG의 한계 (검색 품질 = 답변 상한선)
· RAG vs 파인튜닝, 뭘 선택할까

전체 글은 프로필 링크에서 👉 "RAG란 무엇인가"
🔗 agenwiki.vercel.app/guides/what-is-rag
```

**해시태그:**
```
#RAG #RAG란 #검색증강생성 #agenwiki #LLM개념 #AI공부 #벡터데이터베이스 #임베딩 #인공지능 #AI입문
```

## 세트 3 — "AI한테 시켰는데 답이 왜 이래? — 프롬프트 4가지 기법"

**이미지 순서:** `set3/01-cover` → `02-body-role-context` → `03-body-few-shot` → `04-body-step-by-step` → `05-body-output-format` → `06-cta`

**캡션:**
```
같은 질문인데, 왜 저 사람 답변만 더 좋을까? 🤔

차이는 "프롬프트"입니다. 기법 4가지만 알아도 결과가 확 달라져요 👇
① 역할·맥락 지정 — "너는 5년차 개발자다…"
② 예시 제시(few-shot) — 원하는 형태를 먼저 보여주기
③ 단계 쪼개기 — 한 번에 말고 순서대로
④ 출력 형식 고정 — 표·JSON·불릿으로 지정

각 기법의 나쁜 예 vs 좋은 예를 카드로 정리했어요.
전체 정리와 자주 하는 실수는 프로필 링크에서 👉 "프롬프트 엔지니어링 하는 법"
🔗 agenwiki.vercel.app/guides/prompt-engineering-how-to
```

**해시태그:**
```
